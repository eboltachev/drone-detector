# Crowdsourced Acoustic UAV Detection — безопасный demo-стенд

Данный проект — веб-приложение для презентации бизнес-идеи и технической архитектуры краудсорсингового акустического обнаружения БПЛА. Это **безопасная демонстрационная имитация**: все аудио-события, координаты, классификация, confidence и траектории являются синтетическими.

> Данный стенд является демонстрационной имитацией. Он не предназначен для оперативного обнаружения, сопровождения, наведения, перехвата или поражения БПЛА. Все данные синтетические. Результаты классификации и траектории являются условными.

## Запуск

```bash
docker compose up --build -d
```

Веб-интерфейс будет доступен по адресу:

```text
http://localhost:6001
```

Остановка и очистка томов:

```bash
docker compose down -v
```

## Архитектура

- **frontend**: операторская панель с открытой картой OpenStreetMap района Екатеринбурга в центре России; точки сообщений строятся по их синтетическим координатам рядом с местами сообщений. Сенсоры и ручное добавление событий в интерфейсе отключены.
- **backend**: Python 3.12, FastAPI, Pydantic, SQLAlchemy. Отдает REST API и Server-Sent Events `/api/stream` для обновлений интерфейса в реальном времени.
- **postgres**: PostgreSQL 16 для хранения сенсоров, синтетических аудио-событий и текущего инцидента.
- **ml_simulator.py**: имитационный классификатор, который возвращает `drone_like`, `vehicle_noise`, `wind_noise` или `unknown` и снижает confidence при плохом качестве записи.
- **fusion.py**: демонстрационный fusion-модуль, который агрегирует последние сообщения, строит условную траекторию, общий confidence и направление движения. Он не рассчитывает точное местоположение объекта.

## API

- `GET /api/health` — healthcheck и подтверждение demo-режима.
- `GET /api/sensors` — служебный список синтетических источников для сценария; в UI сенсоры не отображаются.
- `GET /api/events` — журнал аудио-событий.
- `POST /api/events` и `POST /api/events/upload` — служебные endpoints backend; ручное добавление событий в операторском UI отключено.
- `GET /api/incidents/current` — текущая демонстрационная гипотеза.
- `POST /api/scenario/start` — запуск автосценария с постепенным поступлением событий.
- `POST /api/scenario/reset` — очистка событий и сброс инцидента.
- `GET /api/stream` — Server-Sent Events для обновления карты и карточек.

## Синтетический сценарий

Кнопка **«Запустить сценарий»** генерирует последовательность событий от разных сенсоров. Часть событий имитирует транспорт или ветер, часть — условный класс `drone_like`. По мере накопления сообщений:

1. сообщения появляются на открытой карте OpenStreetMap в районе Екатеринбурга с кругами доверия;
2. качество записи подсвечивается зеленым, желтым или красным;
3. fusion-модуль строит презентационную траекторию;
4. confidence и карточка инцидента обновляются через `/api/stream`;
5. оператор видит сообщение «гипотеза построена по N сообщениям» и предупреждение о необходимости подтверждения дополнительными источниками.

## Ограничения безопасности

Проект **не является реальной системой обнаружения** и не должен использоваться для оперативных задач. В нем намеренно не реализованы:

- реальная обработка боевых аудиоданных;
- скрытая запись микрофона;
- сбор персональных данных;
- точное сопровождение цели;
- наведение, перехват или поражение;
- передача данных в военные, силовые или сторонние оперативные системы;
- инструкции по повышению точности реального обнаружения.
